Clipman Android Test Build Manual

Clipman Android is a foreground Clipman Server client for Android phones and tablets. It lets you browse, search, copy, add, edit, pin, delete, and open links from a shared Clipman Server text history while the app is open.

This Android package is currently a test build. It is intended for Clipman Server users who want to help test the Android client before a wider Android release.

Requirements

  • Android 8.0 or newer.
  • A configured Clipman Server.
  • The Clipman Server address and token.
  • The same history password used by your Windows or Mac Clipman clients.
  • Biometric unlock or Android device credential, such as fingerprint, face unlock, PIN, pattern, or password.

Clipman Android does not monitor the Android clipboard in the background. Android restricts background clipboard access, and Clipman follows that platform rule. The app syncs while it is open.

Install

  1. Copy the APK to the Android device.
  2. Open it from Files, Downloads, or your preferred file manager.
  3. If Android asks whether to allow installing unknown apps from that source, allow it for this install.
  4. Install Clipman.
  5. Open Clipman.

The Android package ID is me.onj.clipman.

First Run

On launch, Clipman asks Android to unlock with biometrics or your device credential before it shows history or settings.

The first time you configure it:

  1. Open Settings if it is not already shown.
  2. Enter the Clipman Server address.
  3. Enter the server token.
  4. Enter the history password.
  5. Choose Save and Close.

Once configured, the server connection section is collapsed in Settings. You can open it again with Show server connection.

Main Screen

The main toolbar includes:

  • Text or Links: switches between ordinary text history and link history.
  • Paste: adds the current Android clipboard text to Clipman history.
  • Group: filters entries by group.
  • Settings: opens the full-screen Settings screen.
  • Top and Bottom: jump through long history lists.
  • Sort: cycles sort modes.

Each history row is one accessibility focus target. TalkBack users can move from entry to entry with a single swipe.

Double-tap an entry to copy it to the Android clipboard. Clipman announces Copied to clipboard and plays the copy sound when sounds are enabled.

Long-press, or use TalkBack actions, to open View entry. TalkBack actions also include Edit entry, Pin entry or Unpin entry, and Delete entry.

View Entry

View entry shows the clipboard text line by line so TalkBack can review it naturally.

If Clipman detects one link in an entry, the history row exposes an Open link action.

If Clipman detects multiple links in an entry, View entry lists each URL after the text. Each link is its own button and speaks the URL itself, so you know where it goes before opening it.

Settings

Settings includes:

  • Copy remote additions to Android clipboard: when another machine adds new text to the shared history, Clipman can place that new text on the Android clipboard while the app is open.
  • Play sounds: enables Clipman sounds.
  • Use haptic feedback: vibrates for supported feedback events.
  • Server connection: shows or hides server address, server token, history password, and show-password controls.
  • Build information: shows the app version, readable UTC build time, and exact build stamp so a tester can identify the installed build unambiguously.

If only some custom sounds are available, Clipman uses those and falls back to bundled defaults for the rest.

Server Addresses

Server addresses may be entered as:

clipman://host:port
host:port
http://host:port
https://host:port

clipman:// is Clipman's private-network server scheme. Use HTTPS or a VPN such as Tailscale when connecting across untrusted networks.

Privacy and Security Notes

  • Clipman requires Android biometric or device unlock before showing history or settings.
  • Server address, server token, and history password are stored using Android protected app storage.
  • Clipman Android only syncs while the app is open.
  • The Android clipboard itself is controlled by Android. Once an entry is copied to the clipboard, other apps may be able to paste it according to Android's normal clipboard rules.
