Clipman: Accessible Clipboard Management Tool

Clipman is a small accessible clipboard management tool for Windows and macOS, designed for fast keyboard and screen-reader use. It keeps a searchable clipboard history and can store that history in a cloud service or network share. Add, remove, move, rename, group, pin, or edit text entries on one machine, and other machines watching the same shared database can pick up those changes automatically.

Notes

Clipman runs as a portable Windows app or a Mac menu bar app. It stores machine-specific settings separately from shared clipboard history, so several computers can use the same data folder without sharing hotkeys, startup choices, or local-only preferences.

Basic Use

  • Copy normally. Clipman stores text clips and separate file-history events while monitoring is on.
  • Open history, choose an item, and press Enter to put it back on the clipboard. Press Esc or close the window to hide history without quitting.
  • Use search, groups, pinned entries, sort order, and Entry Properties as your history grows. The detailed commands are in the Keyboard Shortcuts table.
  • Use template entries for reusable text that should resolve date, time, system, or username variables only when copied or pasted.
  • Turn monitoring off temporarily when copying text that should not enter history, add sensitive apps to the ignored applications list, or enable sensitive-data exclusions for known text patterns.
  • On Mac, Option+M opens the history window's Clipman command menu directly.

Preferences

Open Preferences from Options, Preferences on Windows, from the Clipman status menu on Mac, or with Ctrl+, on Windows and Command+, on Mac. Preferences controls monitoring, sounds, hotkeys, storage, history password, ignored applications, optional Links history, startup, update checks, and shared-database behavior. In hotkey fields, press a valid shortcut to assign it, or press Delete or Backspace to clear it.

Quick Paste: Assign Quick Paste from Entry properties or the selected entry's menu. Each assignment is stored in this machine's settings, so several entries can have their own global hotkeys, and each computer can use different Quick Paste assignments even when several computers share the same clipboard database. Quick Paste still works while clipboard monitoring is off, because it is an explicit user command rather than automatic capture.

Each Quick Paste target has a mode. Paste and restore previous clipboard temporarily places the entry on the system clipboard, sends the normal paste command to the active app, and then restores the previous clipboard contents where possible. Paste and keep target on clipboard pastes the entry and leaves that entry on the clipboard. Copy to clipboard only places the entry on the clipboard without sending a paste command.

Use the Quick Paste menu on Windows, or Quick Paste Targets from the Mac Option+M Clipman menu, to review all entries that currently have Quick Paste hotkeys. Choosing one of those menu items moves focus to the matching entry so you can press F2 to edit or remove its Quick Paste assignment. Entries with Quick Paste targets include the assigned hotkey and mode in the row text; entries without Quick Paste targets do not carry that extra wording.

Template entries: In Entry Properties, mark a text entry as a template when you want variables to resolve at copy or paste time. The stored text remains unchanged in the database, exports, search, and Entry Properties. Clipman resolves the template only when the entry is copied, quick-pasted, or automatically copied as a new remote entry.

Template variables include {{year_full}}, {{year_short}}, {{month_name}}, {{month_name_full}}, {{month_name_short}}, {{month_num}}, {{month_num_padded}}, {{day_of_month}}, {{day_of_month_padded}}, {{day_name_full}}, {{day_name_short}}, {{hour_24}}, {{hour_24_padded}}, {{hour_12}}, {{hour_12_padded}}, {{minute}}, {{minute_padded}}, {{second}}, {{second_padded}}, {{utc_offset}}, {{time_zone}}, {{time_zone_short}}, {{os_name}}, {{os_version}}, and {{username}}. Unknown variables are left alone.

Example templates: {{year_full}}-{{month_num_padded}}-{{day_of_month_padded}}, Today is {{day_name_full}}, {{month_name_full}} {{day_of_month}}, {{year_full}}, or {{os_name}} version {{os_version}}. Entry Properties includes buttons to insert sample templates, insert individual fields at the current cursor, preview the resolved template, and view the variable list. Built-in samples include {{year_full}}/{{month_num_padded}}/{{day_of_month_padded}}, {{day_of_month_padded}} {{month_name_short}} {{year_full}}, and {{month_name_short}} {{day_of_month_padded}}, {{year_full}}.

Automatically group new clips by source application is enabled by default for new installs where the setting is exposed. It places newly captured clipboard text into a group named after the application Clipman believes supplied the clipboard text, such as Chrome, Thunderbird, Outlook, Safari, TextEdit, or Logic. Disable it if you prefer all new entries to start ungrouped.

Windows first asks for the clipboard owner application, then falls back to the foreground application if Windows does not expose a useful owner. Mac uses the foreground application reported by macOS when the clipboard change is captured. Some background hotkeys or automation tools may still be attributed to the foreground application because operating systems do not always preserve reliable source metadata.

Global hotkey fields accept combinations with modifiers and one real key, such as Ctrl+Alt+\, Ctrl+Shift+H, Option+Shift+1, or Control+Option+F1. Two modifiers are safest. For compatibility with existing clipboard-manager habits, Clipman also allows one modifier with function keys or with the grave/accent and backslash-style punctuation keys. Single-modifier letters, numbers, comma, space, Tab, Delete, Backspace, Escape, and reserved operating-system shortcuts are rejected. Clipman warns before saving a single-modifier global hotkey because it may conflict with another app or keyboard layout.

Storage and Password: Choose the shared Clipman data folder, set or change the history password, choose whether this computer remembers the password, remove password encryption, and manage ignored applications. Clipman stores shared text history as clipman-history.clipdb inside the chosen folder.

The Remember history password on this computer checkbox controls convenience versus exposure. When unchecked, Clipman asks for the history password when it starts and keeps the password only for the current run. When checked, Windows stores an unlockable password protected by Windows for the current user and machine, and Mac stores it in that user's Keychain for the selected database path. That is convenient for automatic startup, but it cannot protect against malware already running as the same signed-in user. The Use no password button clears the saved history password, clears any session-only password, and rewrites the current database without password encryption. It exists because blank password fields mean "keep the current password"; the explicit button prevents accidental decryption when you only meant to change another setting.

Sensitive data: Sensitive-data exclusions are off by default and machine-specific. When the mode is set to Exclude from history, Clipman checks automatic clipboard text capture against the presets you enable. Matching text is not stored in Clipman history, and the system clipboard is not changed, so your immediate paste still works. This filter does not apply to existing history, imports, Send To, Quick Paste, Push to other machines, or entries manually copied from Clipman.

The built-in presets include credit card numbers with Luhn validation, US Social Security numbers, international phone numbers, long API keys or tokens, software license keys, and approximate US driver-license shapes. International phone numbers include compact E.164-style numbers such as +447890123456 and common spaced, dashed, dotted, or bracketed variants. Built-in presets are deliberately unchecked by default because pattern matching can have false positives.

Platform Notes

The same manual ships with the Windows and Mac builds. The app name, shared history format, sounds folder behavior, password model, and database sync model are intended to stay aligned across platforms.

Windows: Clipman runs in the notification area and opens the history window with the configured global hotkey. Windows Preferences includes Send To integration, Windows startup registration, command-line import/export support, and Windows-protected password remembering when explicitly enabled.

Mac: Clipman is packaged as Clipman.app and runs as a menu bar app. Release builds may be unsigned or ad-hoc signed; after unzipping, move or drag Clipman.app to /Applications. If macOS blocks the first launch, Control-click or Option-click the app and choose Open. With VoiceOver, use VO+Shift+M on the app in Finder for the same context menu, then choose Open.

The Mac status menu includes Show History, Show File History, Toggle Monitoring, Secrets, Open Manual, Check for Updates, Version History, Project Page, Contact, Donate, Diagnostics, Preferences, About, and Quit. The history window also has a tabbable Clipman menu button because menu bar access can be less reliable for an accessory app. Press Option+M in the history window to open that menu directly. Mac Preferences uses Keychain only when password remembering is explicitly enabled.

Mac VoiceOver notes: Clipman is an accessory menu bar app, so it may not always appear in the normal app switcher like a full document app. Use the Show History global hotkey to bring the history window forward. If VoiceOver reports that the system has a new window but focus lands badly, press the Show History hotkey once to dismiss Clipman and press it again to reopen with a fresh focus attempt. In the history window, Option+M opens the Clipman command menu, Control+1 switches Text history, and Command+F opens search. With Links history disabled, Control+2 switches File history. With Links history enabled, Control+2 switches Links history and Control+3 switches File history. Pressing Enter from search returns focus to the history list.

Mac global hotkeys are stored by physical key code so the displayed shortcut remains readable on different keyboard layouts. Default Mac global hotkeys avoid common VoiceOver conflicts, but they can be changed in Preferences.

Startup State

Clipman remembers whether monitoring was on or off. When sounds are enabled, it plays the on or off sound at launch so you know the app is running and which state it restored.

Use Preferences, Startup and updates, Run Clipman at Windows startup on Windows or Run Clipman at login on Mac to start Clipman automatically when you sign in. Windows creates a per-user startup entry for the current clipman.exe path. Mac creates a per-user LaunchAgent for the current Clipman.app. Neither requires administrator permission.

Use Add current clipboard item to Clipman on start if you want Clipman to try to save whatever is already on the system clipboard when Clipman launches. This is off by default. When enabled, it still follows normal monitoring state, ignored-application rules, clipboard privacy signals where supported, concealed pasteboard handling where supported, and Sensitive Data preferences.

Pinned Entries

Pinned entries are grouped at the top of the history list. A separator marks the start of normal entries. Pinned entries are numbered when they can be copied by shortcut, so the first pinned entry starts with 1, the second with 2, and so on through 0 for the tenth. Pinned state is still shown in the Pinned column or metadata, but not before the item text.

Pinned entries are protected from normal cleanup: Delete skips them, and history size or age limits do not remove them. Use Shift+Enter or the menu command to change the pinned state. Use the pinned-entry shortcuts in the Keyboard Shortcuts table to copy one of the first ten pinned entries and close history.

Entry Names and Text

Use F2 to open Entry Properties for the selected entry. You can add a short name or hint, assign a group, configure a Quick Paste hotkey, and edit the stored clipboard text itself. This is useful when a clip contains extra lines or unwanted surrounding text and you want to clean it without using another editor.

In Entry Properties and the F4 text review window, word navigation is tuned for URLs and code-like strings. On Windows, Ctrl+Left and Ctrl+Right stop at punctuation and symbol boundaries such as slashes, question marks, ampersands, equals signs, dashes, and underscores. On Mac, use Option+Left and Option+Right. Add Shift on either platform to extend the selection.

For example, an entry containing 1234 can be shown as Test: 1234. The name is metadata only: copying, cutting, exporting to text, and viewing full text use the clipboard text without the name.

Type-to-jump navigation uses the original clipboard text, not the entry name. If 1234 is named Test, typing 123 jumps to that entry, while typing Test does not jump to it because the clipboard text does not start with Test. Search with Ctrl+F checks both names and clipboard text.

For quick named lookup, type a slash followed by the entry name prefix, such as /Test. Clipman jumps to the matching named entry when it is visible in the current group filter.

Links History

Links history is optional and off by default. Enable it in Preferences with Show Links history tab. When enabled, Clipman adds a separate Links history view for entries where the whole clipboard text is a single absolute http or https URL. Prose that merely contains a link, multiline text, mailto:, file:, and custom schemes remain in Text history.

Links are not stored in a separate database and are not removed from shared text history. The Links tab is a filtered view of the same entries, so copy, cut, paste after selected, grouping, pinning, Quick Paste, templates, search, sorting, cleanup actions, and sharing across machines work like Text history wherever applicable. If the Links tab is disabled, links simply appear in Text history as before.

With Links history disabled, File history remains the second history area. With Links history enabled, Links history becomes the second area and File history moves to the third. On Mac this means Control+2 opens Links and Control+3 opens Files only when Links is enabled. On Windows, Alt+L opens Links when enabled, Alt+I opens File history, and Ctrl+Tab or Ctrl+Shift+Tab moves through the visible history areas.

Groups

Use Ctrl+G on Windows, Command+G on Mac, or the group command in the history window to assign selected entries to a group. Type a new group name, or choose an existing group from the list and press Enter or OK. Choose No group, or leave the group name blank, to remove the group. The Group filter can show all entries, pinned entries, named entries, ungrouped entries, or one of your custom groups.

On Windows, the Groups menu lists the same filters as the Group field. The first ten filters show their Alt+1 through Alt+0 shortcuts in the menu; additional groups appear by name without a numbered shortcut. The top-level Groups menu deliberately does not use Alt+G, because Alt+G moves focus directly to the Group field.

Use Alt+1 through Alt+0 on Windows, or Option+1 through Option+0 on Mac, to jump to one of the first ten group filters. These follow the same order as the Group filter field: All, Pinned, Named, Ungrouped, then custom groups.

On Windows, the Group filter applies when you press Enter in the group field or move focus away from it. Arrowing through the group list does not immediately throw focus back to the history list. On Mac, open the group filter with Option+G or the toolbar Filter button, then choose the filter from the menu.

Sorting and Manual Order

The Text history list includes the entry text, name, group, machine, added time, and last-used time. The machine column records the computer that added or most recently re-added the entry, which helps when several computers share the same database.

The View menu on Windows includes Text history, Links history, and File history commands with Alt+T, Alt+L, and Alt+I. The View menu on Windows, and the Sort controls or Clipman menu on Mac, can sort normal Text history entries by last used, added date, text, group, machine, or manual order. On File history, the same controls sort by time captured, file count, name, operation, source application, or manual order.

Sort direction labels describe the next result plainly, such as oldest first, newest first, A first, Z first, fewest files first, or most files first. Pinned entries and pinned file events stay above normal items and keep their own manual order. Move pinned and normal items separately.

Cut, Copy, and Paste Inside Clipman

Ctrl+C and Ctrl+X on Windows, or Command+C and Command+X on Mac, put normal text on the system clipboard, so the text can be pasted into other applications. Windows also stores a private clipboard format at the same time, so if multiple entries are copied or cut and then pasted back into Clipman with Ctrl+V, they return as separate history entries. Mac pastes multiple copied entries back as separate entries when they are separated by Clipman's text separator.

History Limits

Preferences can limit the maximum number of entries or maximum entry age. Use 0 for no limit. Pinned entries and named entries are kept even when limits are applied.

Actions

On Windows, the Actions menu contains simple text cleanup tools for selected entries: copy as plain text, trim whitespace, convert to single line, remove blank lines, remove URL tracking, clean links for sharing, line endings, uppercase, lowercase, HTML encode, HTML decode, HTML to readable text, URL encode, and URL decode. The most common actions also have shortcuts: Ctrl+Shift+C for plain-text copy, Ctrl+Shift+T for trim, Ctrl+Shift+L for single line, Ctrl+Shift+B for remove blank lines, Ctrl+Shift+R for remove URL tracking, Ctrl+Shift+S for clean link for sharing, Ctrl+Shift+H for HTML to readable text, and Ctrl+Shift+U for URL encode. Transform actions update the stored entry and also put the transformed result on the Windows clipboard. Mac includes URL cleanup and line-ending actions in the Clipman menu; Command+Shift+R removes URL tracking and Command+Shift+S cleans links for sharing. These update the stored entry and put the transformed result on the Mac clipboard.

On Windows, Copy as plain text copies the selected entry using only the Windows plain-text clipboard format. It does not strip HTML tags from stored HTML source. Use HTML to readable text when you want Clipman to remove HTML tags and keep the readable text.

Remove URL tracking removes common tracking query parameters from web links, including utm_*, fbclid, gclid, msclkid, and similar marketing identifiers. It works on plain links and common HTML link attributes such as href and src. Automatic URL-tracking removal is opt-in from Preferences; when enabled, newly copied text is cleaned before Clipman stores it, and the cleaned text is placed on the Windows clipboard.

Clean link for sharing removes tracking parameters and also removes share-state parameters that can change how a shared link opens. For YouTube links this includes timestamp parameters such as t, start, and time_continue, plus common share metadata such as pp, si, and feature. Use this when you want to share the clean video or page URL rather than a link that opens from the middle of a video.

Line endings normalizes selected text entries to Windows CRLF, Unix LF, or old Mac CR. Use Windows CRLF when text copied from macOS, Linux, web tools, or scripts appears as one long line in an older Windows editor. This is an explicit action rather than an automatic default, because code, scripts, patches, and configuration files may need their original line endings preserved.

URL encode is for text that needs to be safe inside a URL. For example, Where's My Car? becomes Where%27s%20My%20Car%3F. HTML encode is for text that needs to be safe inside HTML source, where characters such as ampersands and quotes may need to become HTML entities. Use URL encode for web links, query strings, podcast episode links, and similar copy/paste jobs.

Sensitive Data Exclusions

Sensitive-data exclusions live in Preferences. They are off by default. Set the mode to Exclude from history, then check the presets you want. When automatic clipboard text matches a checked preset, Clipman does not save that text to history. The system clipboard or Mac pasteboard is not changed, so your immediate paste still works.

This feature is deliberately conservative. It applies only to automatic text capture. It does not filter existing history, imports, Send To, Quick Paste, Push to other machines, or entries manually copied from Clipman. Presets can have false positives, so enable only the ones that fit the work you do.

Clipboard Privacy Signals

On Windows, Clipman also honors standard clipboard privacy signals added by the application that placed data on the clipboard. If a clipboard update contains Clipboard Viewer Ignore, ExcludeClipboardContentFromMonitorProcessing, CanIncludeInClipboardHistory set to zero, or CanUploadToCloudClipboard set to zero, Clipman treats that update as private, plays exclude.wav when sounds are enabled, and stores nothing in text history or file history.

This helps with password managers and secure-copy workflows that mark their own clipboard data as not suitable for history or sync. It is still cooperative: if Chrome, a webpage, a password manager, or another app puts only ordinary text on the clipboard with no privacy format, Clipman cannot reliably tell whether that text came from a password field or a normal webpage. Use ignored applications and Sensitive Data Exclusions for cases where the source app does not mark private clipboard updates itself.

Ignored Applications

Preferences can list applications that Clipman should ignore, one per line. This is useful for password managers, banking tools, private notes, or any app where clipboard history should not be recorded.

On Windows, enter process names with or without .exe, or use Add running app to choose from currently running processes. On Mac, enter the app name, bundle identifier, or executable name, such as Safari, com.apple.TextEdit, or KeePassXC. Ignored applications are machine-specific settings. Clipman also treats related helper names as ignored when they use the same name or bundle/process prefix. On Mac, Clipman also refuses concealed pasteboard data and can use pasteboard type metadata from password-manager quick-access panels where macOS does not report the password manager as the foreground app.

Ignored applications and sensitive-data exclusions solve different problems. Ignored applications are best when an entire app should never be captured. Sensitive-data exclusions are best when ordinary apps sometimes copy text matching a known sensitive pattern.

Database and Sync

On Windows, Clipman defaults to a portable Settings folder beside clipman.exe. On Mac, Clipman stores a small pointer under Application Support and keeps the active machine settings in the selected Clipman data folder. Settings are machine-specific and use the computer name, such as Settings\Desktop-settings.json, Settings\Laptop-settings.json, or MacBook-settings.json. This lets multiple computers share one data folder without fighting over hotkeys, monitoring state, startup choices, or window position. The shared text history database is clipman-history.clipdb.

If you choose a different data folder, Clipman stores this machine's active settings in that folder beside clipman-history.clipdb and the machine-specific file-history database. A small settings-location.json pointer remains in the app's local Settings folder so Clipman can find the selected data folder on the next launch. That pointer stores locations per computer name, so two machines using different local paths to reach the same cloud or network data folder do not overwrite each other's location. If a cloud service creates a conflict copy of the pointer, Clipman merges the known clients back into one pointer file.

If the default data folder is in use, Clipman treats it as portable and relocates it when the app folder is moved. If you choose or type a different data folder, Clipman stores the shared history as clipman-history.clipdb inside that folder. Existing explicit .clipdb file paths remain readable for compatibility. If an explicit data location is missing at startup, Clipman asks whether to browse for the data folder in a new location, use the default database beside Clipman, or stop startup.

The live history database uses Clipman's compressed .clipdb format. By default, compression keeps clipboard text out of casual plain-text searches by Windows Search, Spotlight, cloud sync indexers, archive tools, or similar tools. To share clipboard history between machines, set each machine's data folder to the same synced or network-shared folder. Clipman uses the clipman-history.clipdb file inside that folder for shared text history.

Multiple running Clipman instances can use the same history database. When one machine writes a new entry, the database is saved atomically and the other machines reload it when the changed file arrives. Each text entry stores the machine name that added or most recently re-added it, and the history list includes a Machine column and machine sort mode so shared setups can tell which computer supplied an entry.

Preferences can optionally put newly created text entries received from another machine onto this machine's clipboard. This is off by default. When enabled, Clipman baselines the current newest remote entry first, so enabling the option does not unexpectedly copy older history. Reusing an older entry with Quick Paste on another machine updates that entry's last-used time, but it does not trigger remote auto-copy because it is not a newly created entry. To intentionally send an existing selected entry to other machines, use Push to other machines from the entry menu, or press Ctrl+P on Windows and Command+P on Mac. This creates a fresh shared-history event from the selected text so other opted-in machines can receive it as the latest remote text. When duplicate removal is enabled, Clipman refreshes the selected entry instead of leaving an extra duplicate behind. This preference is machine-specific and does not change the shared database format.

If the shared database file is missing but its folder is available, Clipman starts with an empty history and creates the file when it next saves. If the shared folder or drive is temporarily unavailable, Clipman keeps running, reports the storage problem in diagnostics, and avoids crashing. Background clipboard captures do not display blocking error dialogs while storage is missing. On Mac, Clipman pauses monitoring and adds a Retry Storage command to the status menu until the data folder returns. On Windows, the notification-area menu and tooltip report that storage is unavailable and offer Retry storage. When the location becomes available again, Clipman merges the existing database before saving so it does not blindly overwrite a shared history file.

History password: Preferences can encrypt the shared history database with a password. On a new database, leaving the password fields blank means Clipman uses compressed .clipdb storage without password encryption. If you enter and confirm a history password, the database is encrypted with AES and checked with a message authentication code before loading. The password is not stored inside the shared .clipdb file.

New settings do not remember the history password by default. In that mode, Clipman prompts for the password when it starts, keeps it only while Clipman is running, and clears any saved platform password. If you check Remember history password on this computer, Windows stores the password using Windows user protection for this account and machine, and Mac stores it in that user's Keychain for the selected database path. On Mac, Preferences reports whether database encryption is on and whether the password is saved in Keychain; the password field stays blank for security even when a remembered password exists. Copying a settings file to another computer does not copy a working key; that computer must be given the same history password through its own Preferences window. Remembered passwords are still recoverable by software already running as the same signed-in user, so leave Remember unchecked if you prefer startup prompts over same-user recoverability.

The Generate password button, where available, creates a random history password, fills both password fields, and copies it to the clipboard. Clipman deliberately ignores that generated password copy, so it is not added to clipboard history. You are responsible for saving that password somewhere safe and entering it on each computer that shares the encrypted history. If the password is lost, Clipman cannot recover the encrypted history.

When typing a history password manually, enter it in both password fields. The Show history password checkbox temporarily reveals both fields so you can verify what you typed. Leave both password fields blank to keep the current encrypted database password. Use the Use no password button if you want to remove password encryption and rewrite the current database without a history password. Windows command-line import, export, and Send To operations can open encrypted databases only when this computer is set to remember the password; the already-running Clipman window can use a session-only password.

If a sync service creates conflict copies of Clipman's settings or history database, Clipman attempts to tidy them automatically when it next loads the affected file. History database conflicts are merged by entry, then the conflict copies are removed. Machine settings conflicts use the newest copy for that machine, then remove the extra files. This is intended only for Clipman's own settings and .clipdb files.

Clipman also writes a small shared update-state file in the Settings folder. It records the current public version, an internal UTC build stamp, the machine that published it, and the expected executable hash. If another running Clipman instance sees a newer build stamp and the updated executable has synced to that machine, it restarts itself so it runs the current copy.

During an online or automatic update, Clipman downloads the release ZIP to a temporary folder, asks other instances running from the same shared folder to close briefly, replaces the app files while preserving Settings, then starts the updated copy. When the new copy starts it publishes the new shared update state, so other computers can restart after their cloud service or network share exposes the updated executable. This is designed to make shared-folder updates feel seamless whether the folder is synced locally or opened from a direct network share.

Secrets

Clipman 2.0 includes a small optional Secrets area for short private snippets that you type deliberately, such as a system password or code that you sometimes need to paste quickly outside a web browser. Secrets are not captured automatically from the clipboard, are not shown in normal Text history, and are not included in ordinary history import or export. Open Options, Secrets on Windows, Secrets from the Mac status menu, or Secrets from the Mac in-window Clipman menu with Option+M. You can also open Secrets directly with Ctrl+Shift+E on Windows or Command+Shift+E on Mac.

Secrets require a history password. They are stored in a separate per-machine <computer-name>-secrets.clipdb file beside the active settings/history files and use the same history password as the main database. They do not sync through the shared text-history database. Opening the Secrets manager asks for the current history password so names and assigned hotkeys are not shown casually from an unlocked desktop. The secrets list shows names and assigned hotkeys only; secret values are hidden in the list and are edited in password fields. If you remove the history password, Clipman does not rewrite secrets as an unencrypted file.

In the Secrets manager, press Insert to add a secret, F2 to edit the selected secret, Delete to remove it, Enter to Quick Paste it immediately, and Esc to close the window. The secret editor has fields for a name, the secret value, confirmation, and an optional Quick Paste hotkey. Ctrl+A on Windows and Command+A on Mac work in the password and secret text fields so a mistyped value can be selected and replaced without holding Backspace.

Each secret can have its own Quick Paste hotkey. Pressing that hotkey temporarily places the secret value on the system clipboard, sends the normal paste command to the active app, then restores the previous clipboard where the platform allows it. This is meant for convenience, not as a full password manager: any app that can monitor keystrokes, inspect the clipboard at the wrong moment, or run as the same signed-in user may still be able to see what you paste.

Clipman Server

Clipman 2.0 adds an optional Clipman Server path for people who want to host the shared history database from one machine, a Raspberry Pi, or a VPS instead of relying only on a cloud service, synced folder, or network share. This is an additional storage path, not a replacement for the existing local and shared-folder workflow.

The normal Clipman app packages are client-only. Clipman Server is distributed as a separate cross-platform server package from the Clipman releases page. That package runs the same Python server on Linux, macOS, and Windows so the safety model stays consistent: random high port and random bearer token on first run, password-scoped database buckets, cheap revision polling, rolling backups, private file permissions where supported, rotating logs, database-ID log redaction, lightweight runtime counters, and optional direct TLS.

The server stores and serves raw .clipdb files. It does not know, request, or store the history password. Encrypted databases remain encrypted on the server; Clipman clients unlock, merge, and save history locally, then upload the updated database with a revision check. If another client has already uploaded a newer copy, the server rejects the stale upload so the client can download, merge, and try again instead of overwriting newer history.

Each server token plus each history password maps to a separate server-side database bucket. That means two clients using the same server token and same history password see the same history, while a wrong password or intentionally different password does not overwrite the existing database. No-password histories are also separated from password-protected histories.

Backups are written under each database bucket. The default backup schedule is hourly, with roughly 24 hours of backups kept and a maximum of 48 backup files.

For private LAN, localhost, or VPN-only use, clients can use clipman://host:port. Clipman displays that form to avoid making the server look like a normal web page, then uses the local server transport internally. Do not expose plain clipman:// or plain HTTP to the public internet. For VPS or internet-facing use, put the server behind HTTPS or start the server with --cert-file and --key-file so clients connect using an https:// address.

On Linux, the server package includes Linux/install-clipman-server.sh. The installer copies the server into a user-local application folder, creates a launcher, creates first-run settings, and writes clipman-server-connection.txt beside the settings file. That text file contains the server address, port, and token so users do not have to copy values from JSON. Delete it or move the details to a password manager once every client is configured.

To start a server from an existing history, connect first from the Clipman client that already has the desired history and history password. Do not manually copy a .clipdb into the server's database folders; clients derive the correct server bucket from the server token and the history password, and the server deliberately does not know the history password.

To use a server from the Windows or Mac client, open Preferences, set Storage type to Clipman Server, enter the server host and token, and keep the data folder set to a local cache/settings folder. The host can be typed as home-server:49152; Clipman infers the local server protocol. The server token field is hidden on screen and saved with per-user protection: Windows uses current-user DPAPI protection, and Mac uses the user's Keychain. The client still watches and saves a local cache, but text history is synchronized through the server.

Clients normally poll the server cheaply by checking the database revision. If the server, network, or VPN is temporarily unavailable, Clipman keeps its local cache, backs off repeated polling failures, and retries automatically. Diagnostics include the last server poll, last successful sync, last upload, current server revision, and failure count so sync problems can be diagnosed without guessing.

Read the server package's Manual.html for setup, security, service-path, TLS, logging, backup, and API details.

Send To

Send To is a Windows feature. Windows Preferences can add Clipman to the Windows Send To menu. Sending a supported text-oriented file to Clipman adds the file's text content to clipboard history. Supported examples include plain text, logs, Markdown, CSV, JSON, XML, HTML, CSS, JavaScript, PowerShell, batch files, INI/config files, AutoHotkey scripts, registry files, and common source-code files. The imported text uses the current duplicate and history-limit preferences.

The Send To shortcut is named Clipman - add text to history with C as its menu accelerator.

The Send To entry calls Clipman with the selected file path. If Clipman is already running, the file is imported into the configured database. Preferences can choose whether a successful Send To import also brings the history window to the front. If Clipman is not already running, it imports the file using the settings in the Settings folder beside clipman.exe. When automatic grouping is enabled, Send To imports are placed in a group named Send To. Mac does not currently ship a Send To equivalent.

File History

The File history tab shows file, folder, and non-text clipboard events captured by Clipman. File history is stored separately from text history in a machine-specific database named like Settings\Desktop-file-history.clipdb, Desktop-file-history.clipdb inside a custom data folder, or MacBook-file-history.clipdb inside the selected Mac data folder. If you set a history password, file history uses the same password; there is not a second password to remember. File history stores paths and clipboard-format details only. It does not copy or preserve the actual file contents.

Copying the same file set again moves the existing event to the top instead of adding another duplicate. File history rows start with the file or folder name, then show the operation and file count. Press Enter to restore all existing files and folders from the selected file events to the clipboard and close the history window, matching Text history behavior. Press Ctrl+C on Windows or Command+C on Mac to copy all selected file paths as text without closing. Use the View menu on Windows, or the Sort controls and Clipman menu on Mac, to sort normal file events by time captured, file count, name, operation, source application, or manual order. Press Home, End, Page Up, and Page Down for larger list jumps. Press Backspace to jump to the first normal file event below pinned file events. Press Shift+Enter to pin or unpin selected file events. Press Ctrl+Enter on Windows or Command+Enter on Mac to go to a single selected file or folder in Explorer or Finder. Press Alt+Up or Alt+Down on Windows, or Option+Up or Option+Down on Mac, to move selected file events within the pinned or normal section. Moving file events switches File history back to manual order. Press Del on Windows or Command+Backspace on Mac to remove selected unpinned file-history events from the list.

Pinned file events are grouped at the top of File history, with the same normal-entries separator used by Text history. They are numbered when they can be restored by shortcut, and their pinned state is shown in the Pinned column or metadata rather than before the event name. They are protected from delete, clear-normal cleanup, automatic unavailable-event cleanup, and file-history size trimming. Use Ctrl+1 through Ctrl+0 on Windows, or Command+1 through Command+0 on Mac, from the File history tab to restore one of the first ten pinned file events and close history. Windows also supports Ctrl+Shift+1 through Ctrl+Shift+0 to copy paths from one of the first ten pinned file events as text and close history.

File history can clear normal file-history events or remove unavailable unpinned events. Unavailable events include non-file clipboard events that cannot be restored as files, and file events where all referenced files and folders are now missing. These actions are also available from the keyboard with Ctrl+Del and Alt+Del on Windows, or Control+Backspace and Option+Backspace on Mac. Windows Preferences can also remove unavailable unpinned events automatically as new file-history events are captured.

Import and Export

  • Import adds entries from a .clipdb, JSON, or text file on both platforms. Windows can also import old Clipman clipman.db or Ditto SQLite databases.
  • Import and replace replaces the current clipboard database with the selected file on Windows. Mac currently imports into the existing history.
  • Export saves the current clipboard history as compressed .clipdb, JSON, or text on both platforms.

When importing an encrypted .clipdb, Clipman asks for that import file's password if it is different from the current database password. When exporting a .clipdb from the app, Clipman asks whether to use the current history password, a new export-only password, or no password. If the current history database is password protected, Clipman asks you to re-enter the current history password before creating any .clipdb export, including an export that uses no password. JSON and text exports are deliberately readable interchange formats, so use .clipdb when exporting private history that should stay encrypted.

Old Clipman and Ditto imports are Windows-only and read text entries only. They do not import images or every custom clipboard format. Imported entries are grouped as Imported from old Clipman or Imported from Ditto so they are easy to review after import.

Sounds

Clipman can play short sounds when text is copied, remote text is copied from another synced machine, monitoring is turned on, monitoring is turned off, a clipboard item is ignored because monitoring is off, or a sensitive-data exclusion blocks automatic capture. Sounds can be enabled or disabled in preferences.

The default sounds are ordinary WAV files. On Windows they live in the sounds folder beside clipman.exe. On Mac they are bundled inside Clipman.app. To keep custom sounds safe across updates, create a sounds folder inside Clipman's active settings/data folder and put any replacement WAV files there. For a default Windows portable install this is usually Settings\sounds beside clipman.exe; if you choose a different data folder, use the sounds folder inside that chosen data folder instead. Clipman checks custom sounds first, then falls back to the bundled/default sounds for any missing file. Keep the same file names: copy.wav, remote.wav, on.wav, off.wav, skip.wav, and exclude.wav. If exclude.wav is missing, Clipman falls back to skip.wav.

Non-text Clipboard Events

Clipman stores text history in the main history database and stores file or non-text clipboard events in a separate machine-specific file-history database. Use the File history tab to review those events. It can show files copied or cut in Windows Explorer or macOS Finder, file-like clipboard operations from other programs, and specialist formats such as audio-editor clipboard data. Clipman cannot retrieve events from before it was running, and file events are useful only while the referenced files still exist at the same paths.

Press Enter on a file event to put its files back on the clipboard, ready to paste into Explorer, Finder, or another file-aware application. Press Ctrl+C on Windows or Command+C on Mac to copy the file paths as text, Shift+Enter to pin or unpin selected events, Del on Windows or Command+Backspace on Mac to remove selected unpinned events from file history, F4 to view event details, or Ctrl+Enter on Windows and Command+Enter on Mac to go to one selected file or folder. Non-file events are informational; they can be reviewed, pinned, moved, or deleted, but they cannot be restored as files. Open diagnostics for a text report of the same events.
