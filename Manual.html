Clipman: Accessible Clipboard Management Tool for Windows

Clipman is a small portable accessible clipboard management tool for Windows, designed for fast keyboard and screen-reader use. It runs from its folder, keeps a searchable clipboard history, and can store that history in a cloud service or network share.

Project page: https://github.com/OnjLouis/Clipman

Notes

Clipman is intentionally portable. It does not require an installer. Machine-specific settings and the default shared database stay in the Settings folder near the executable unless you choose another database path.

Basic Use

  • Use the show history hotkey to show or hide the clipboard history window.
  • The history window has two tabs: Text history and File history. Clipman remembers the tab you used last. Use Ctrl+Tab and Ctrl+Shift+Tab to switch tabs.
  • Choose an entry and press Enter to copy it back to the clipboard and close history.
  • Press Ctrl+C to copy an entry without closing the history window.
  • Press Ctrl+X to cut selected unpinned entries: copy them, then remove them from history.
  • Press Ctrl+V to paste the current clipboard text into history after the selected entry. If the normal entries separator is selected, pasted entries go to the top of the normal entries section. If the clipboard came from multiple Clipman entries, they are pasted back as separate entries.
  • Press F2 to edit an entry. The dialog lets you change the entry name and the stored clipboard text.
  • Press Ctrl+F to search names and clipboard text. Press F3 to find the next match, or Shift+F3 to find the previous match. Searches wrap automatically.
  • Press Ctrl+G to assign selected entries to a group. Use the Group filter after the history list to show all entries, pinned entries, named entries, ungrouped entries, or a specific group.
  • Press Alt+G on the Text history tab to move focus to the Group filter.
  • Press Alt+Enter to open entry properties, where you can change name, group, pinned state, review the full text, copy the full text, or delete the entry.
  • Press F4 to view the full selected text in a read-only edit field.
  • Press Shift+Enter to pin or unpin an entry. Pinned entries are protected from delete and cleanup.
  • Select multiple entries to copy, delete, pin, unpin, or move them together.
  • Use the View menu to sort history by last used, added date, text, group, machine, or manual order. The same menu toggles the current sort between ascending and descending.
  • Press Alt+Up or Alt+Down to move selected entries in manual order.
  • Press Ctrl+1 through Ctrl+0 to copy one of the first ten pinned entries and close history.
  • Press Alt+1 through Alt+0 to jump to one of the first ten group filters.
  • Use the File history tab to review recent file copy or cut clipboard events captured while Clipman has been running. Press Enter on a file event to put those files back on the Windows clipboard. Its Application key menu is separate from the text history menu and only shows file-event actions.
  • Press Esc or close the window to hide history without exiting Clipman.
  • Use the toggle hotkey to turn clipboard monitoring on or off.
  • The notification area icon shows whether monitoring is on or off.

Preferences

Open Options, Preferences from the history window. Preferences are split into tabs for General, Hotkeys, Storage and Password, and Startup and updates. Changes apply immediately; use Close when finished. Clipman remembers the Preferences tab you used last. Press Ctrl+1 through Ctrl+4 to switch directly between the tabs.

Preferences can change the global hotkeys, sound behavior, duplicate handling, saved list position, active state, database path, history limits, ignored applications, Send To integration, automatic URL-tracking removal, Windows startup registration, update checks, silent update installs, and whether Send To imports bring the history window to the front.

Automatically group new clips by source application is enabled by default for new installs. It places newly captured clipboard text into a group named after the application Clipman believes supplied the clipboard text, such as Chrome, Thunderbird, or Outlook. Disable it if you prefer all new entries to start ungrouped.

Clipman first asks Windows for the clipboard owner application, then falls back to the foreground application if Windows does not expose a useful owner. Some background hotkeys or automation tools may still be attributed to the foreground application because the Windows clipboard does not always preserve reliable source metadata.

Global hotkey fields accept only combinations with at least two modifiers and one real key, such as Ctrl+Alt+\ or Ctrl+Shift+H. Modifier-only keys, single keys, and unsafe Windows shortcuts are rejected.

Storage and Password: Use this tab to choose the shared history database path, set or change the history password, generate a random password, remove the saved password, and manage ignored applications. The password fields are intentionally blank when Preferences opens, even if a password is already saved. This prevents someone from opening Preferences and revealing the saved key with Show history password.

The Use no password button clears the saved history password and rewrites the current database without password encryption. It exists because blank password fields mean "keep the saved password" on a machine that already has one; the explicit button prevents accidental decryption when you only meant to change another setting.

Startup State

Clipman remembers whether monitoring was on or off. When sounds are enabled, it plays the on or off sound at launch so you know the app is running and which state it restored.

Use Preferences, Startup and updates, Run Clipman at Windows startup to start Clipman automatically when you sign in. This creates a per-user startup entry for the current clipman.exe path and does not require administrator permission.

Pinned Entries

Pinned entries are grouped at the top of the history list. A separator marks the start of normal entries. Press Home to reach the top of the list, and press Backspace to jump to the first normal entry. Pinned entries are read-only for normal cleanup: Delete skips them, and history size or age limits do not remove them. Use Shift+Enter or Edit, Toggle pinned to change the pinned state. Use Ctrl+1 through Ctrl+0 to copy one of the first ten pinned entries and close history.

Entry Names and Text

Use F2 to edit the selected entry. You can add a short name or hint, and you can also edit the stored clipboard text itself. This is useful when a clip contains extra lines or unwanted surrounding text and you want to clean it without using another editor.

For example, an entry containing 1234 can be shown as Test: 1234. The name is metadata only: copying, cutting, exporting to text, and viewing full text use the clipboard text without the name.

Type-to-jump navigation uses the original clipboard text, not the entry name. If 1234 is named Test, typing 123 jumps to that entry, while typing Test does not jump to it because the clipboard text does not start with Test. Search with Ctrl+F checks both names and clipboard text.

For quick named lookup, type a slash followed by the entry name prefix, such as /Test. Clipman jumps to the matching named entry when it is visible in the current group filter.

Groups

Use Ctrl+G or Edit, Group entry to assign selected entries to a group. Type a new group name, or choose an existing group from the list and press Enter or OK. Choose No group, or leave the group name blank, to remove the group. The Group filter after the history list can show all entries, pinned entries, named entries, ungrouped entries, or one of your custom groups.

Use Alt+1 through Alt+0 to jump to one of the first ten group filters. These follow the same order as the Group filter field: All, Pinned, Named, Ungrouped, then custom groups.

The Group filter applies when you press Enter in the group field or move focus away from it. Arrowing through the group list does not immediately throw focus back to the history list.

Sorting and Manual Order

The Text history list includes the entry text, name, group, machine, added time, and last-used time. The machine column records the computer that added or most recently re-added the entry, which helps when several computers share the same database.

The View menu can sort normal history entries by last used, added date, text, group, machine, or manual order. The sort direction command in the same menu toggles the current sort between ascending and descending; its label changes to show the action it will take next. Group sorting places grouped entries together and leaves ungrouped entries after named groups. Machine sorting groups entries by the computer that added or most recently re-added them. Pinned entries stay above normal entries and keep their own manual order regardless of the selected sort mode or direction. Press Alt+Up or Alt+Down to move selected pinned entries within the pinned section, or selected normal entries within the normal history section. Move pinned and normal entries separately.

Cut, Copy, and Paste Inside Clipman

Ctrl+C and Ctrl+X put normal text on the Windows clipboard, so the text can be pasted into other applications. Clipman also stores a private clipboard format at the same time, so if multiple entries are copied or cut and then pasted back into Clipman with Ctrl+V, they return as separate history entries.

History Limits

Preferences can limit the maximum number of entries or maximum entry age. Use 0 for no limit. Pinned entries and named entries are kept even when limits are applied.

Actions

The Actions menu contains simple text cleanup tools for selected entries: copy as plain text, trim whitespace, convert to single line, remove blank lines, remove URL tracking, uppercase, lowercase, HTML encode, HTML decode, HTML to readable text, URL encode, and URL decode. The most common actions also have shortcuts: Ctrl+Shift+C for plain-text copy, Ctrl+Shift+T for trim, Ctrl+Shift+L for single line, Ctrl+Shift+B for remove blank lines, Ctrl+Shift+R for remove URL tracking, Ctrl+Shift+H for HTML to readable text, and Ctrl+Shift+U for URL encode. Transform actions update the stored entry and also put the transformed result on the Windows clipboard.

Copy as plain text copies the selected entry using only the Windows plain-text clipboard format. It does not strip HTML tags from stored HTML source. Use HTML to readable text when you want Clipman to remove HTML tags and keep the readable text.

Remove URL tracking removes common tracking query parameters from web links, including utm_*, fbclid, gclid, msclkid, and similar marketing identifiers. It works on plain links and common HTML link attributes such as href and src. Automatic URL-tracking removal is opt-in from Preferences; when enabled, newly copied text is cleaned before Clipman stores it, and the cleaned text is placed on the Windows clipboard.

URL encode is for text that needs to be safe inside a URL. For example, Where's My Car? becomes Where%27s%20My%20Car%3F. HTML encode is for text that needs to be safe inside HTML source, where characters such as ampersands and quotes may need to become HTML entities. Use URL encode for web links, query strings, podcast episode links, and similar copy/paste jobs.

Ignored Applications

Preferences can list process names that Clipman should ignore, one per line. This is useful for password managers, banking tools, or any app where clipboard history should not be recorded. Process names may be written with or without .exe.

Use Add running app to choose from currently running processes. This adds the exact process name to the ignored applications list. Apps that are not currently running can still be typed into the text box manually.

Database and Sync

By default Clipman stores user data in a Settings folder beside clipman.exe. Settings are machine-specific and use the computer name, such as Settings\Desktop-settings.json or Settings\Laptop-settings.json. This lets multiple computers run the same Clipman folder without fighting over hotkeys, monitoring state, startup choices, or window position. The default shared history database is Settings\clipman-history.clipdb.

The live history database uses Clipman's compressed .clipdb format. By default, compression keeps clipboard text out of casual plain-text searches by Windows Search, cloud sync indexers, archive tools, or similar tools. To share clipboard history between machines, set each machine's database path to the same synced or network-shared .clipdb file.

Multiple running Clipman instances can use the same history database. When one machine writes a new entry, the database is saved atomically and the other machines reload it when the changed file arrives. Each text entry stores the machine name that added or most recently re-added it, and the history list includes a Machine column and machine sort mode so shared setups can tell which computer supplied an entry.

History password: Preferences can encrypt the shared history database with a password. On a new database, leaving the password fields blank means Clipman uses compressed .clipdb storage without password encryption. If you enter and confirm a history password, the database is encrypted with AES and checked with a message authentication code before loading. The history password is protected by Windows for the current user account and machine. Copying a settings file to another computer does not copy a working key; that computer must be given the same history password through its own Preferences window.

The Generate password button creates a random history password, fills both password fields, copies it to the Windows clipboard, and saves it for this Windows user. Clipman deliberately ignores that generated password copy, so it is not added to clipboard history. You are responsible for saving that password somewhere safe and entering it on each computer that shares the encrypted history. If the password is lost, Clipman cannot recover the encrypted history.

When typing a history password manually, enter it in both password fields. The Show history password checkbox temporarily reveals both fields so you can verify what you typed. Leave both password fields blank to keep the password already saved on this machine. Use the Use no password button if you want to remove the saved password and rewrite the current database without password encryption.

If a sync service creates conflict copies of Clipman's settings or history database, Clipman attempts to tidy them automatically when it next loads the affected file. History database conflicts are merged by entry, then the conflict copies are removed. Machine settings conflicts use the newest copy for that machine, then remove the extra files. This is intended only for Clipman's own settings and .clipdb files.

Clipman also writes a small shared update-state file in the Settings folder. It records the current public version, an internal UTC build stamp, the machine that published it, and the expected executable hash. If another running Clipman instance sees a newer build stamp and the updated executable has synced to that machine, it restarts itself so it runs the current copy.

During an online or automatic update, Clipman downloads the release ZIP to a temporary folder, asks other instances running from the same shared folder to close briefly, replaces the app files while preserving Settings, then starts the updated copy. When the new copy starts it publishes the new shared update state, so other computers can restart after their cloud service or network share exposes the updated executable. This is designed to make shared-folder updates feel seamless whether the folder is synced locally or opened from a direct network share.

Send To

Preferences can add Clipman to the Windows Send To menu. Sending a supported text-oriented file to Clipman adds the file's text content to clipboard history. Supported examples include plain text, logs, Markdown, CSV, JSON, XML, HTML, CSS, JavaScript, PowerShell, batch files, INI/config files, AutoHotkey scripts, registry files, and common source-code files. The imported text uses the current duplicate and history-limit preferences.

The Send To shortcut is named Clipman - add text to history with C as its menu accelerator.

The Send To entry calls Clipman with the selected file path. If Clipman is already running, the file is imported into the configured database. Preferences can choose whether a successful Send To import also brings the history window to the front. If Clipman is not already running, it imports the file using the settings in the Settings folder beside clipman.exe. When automatic grouping is enabled, Send To imports are placed in a group named Send To.

File History

The File history tab shows file and folder clipboard events captured while Clipman has been running. Copying the same file set again moves the existing event to the top instead of adding another duplicate. Press Enter to restore the selected file event to the Windows clipboard and close the history window, matching Text history behavior. Press Ctrl+C to copy the file paths as text without closing.

Import and Export

  • Import adds entries from a .clipdb, JSON, text file, old Clipman clipman.db, or Ditto SQLite database.
  • Import and replace replaces the current clipboard database with the selected file.
  • Export saves the current clipboard history as compressed .clipdb, JSON, or text.

When a history password is saved, .clipdb imports and exports use the current history password. JSON and text exports are deliberately readable interchange formats, so use .clipdb when exporting private history that should stay encrypted.

Old Clipman and Ditto imports read text entries only. They do not import images or every custom clipboard format. Imported entries are grouped as Imported from old Clipman or Imported from Ditto so they are easy to review after import.

Updates

Use Help, Check for updates or Shift+F1 to check GitHub releases. If a newer release includes a Clipman ZIP package, Clipman can download it, close, merge the update into the current folder, preserve the Settings folder, and restart.

Use Preferences, Startup and updates to choose whether Clipman checks for updates never, at startup, hourly, or daily. If Install updates silently when possible is enabled, automatic checks can install a newer ZIP release without prompting. Manual update checks still show the normal update dialog.

Use Help, Version history to review the latest release notes, Help, Project page or Ctrl+F1 to open the GitHub project page, Help, Contact to send questions or feedback, Help, Donate to support the project, and Help, Diagnostics or Alt+F1 to open diagnostics.

Sounds

Clipman can play short sounds when text is copied, monitoring is turned on, monitoring is turned off, or a clipboard item is ignored because monitoring is off. Sounds can be enabled or disabled in preferences.

The default sounds are ordinary WAV files in the sounds folder beside clipman.exe. To keep custom sounds safe across updates, create a sounds folder inside Clipman's Settings folder and put any replacement WAV files there. Clipman checks Settings\sounds first, then falls back to the default sounds folder for any missing file. Keep the same file names: copy.wav, on.wav, off.wav, and skip.wav.

Non-text Clipboard Events

Clipman normally stores only text history. It also keeps a small in-memory list of recent non-text clipboard events, such as files copied or cut in Windows Explorer. Use the File history tab to review those events. File history is session-only: it contains file clipboard events captured while Clipman has been running, and it cannot retrieve file events from before Clipman was started. Press Enter on a file event to put its files back on the Windows clipboard, ready to paste into Explorer or another file-aware application. Press Ctrl+C to copy the file paths as text, or F4 to view event details. The Application key menu on this tab contains only file-event actions. Open Help, Diagnostics for a text report of the same recent events. They are not saved to the history database and are cleared when Clipman exits.

Command Line

Command Effect
clipman.exe --show Shows the history window for the running Clipman instance.
clipman.exe --close Closes the running Clipman instance.
clipman.exe --pause Turns clipboard monitoring off in the running Clipman instance.
clipman.exe --resume Turns clipboard monitoring on in the running Clipman instance.
clipman.exe --toggle Toggles clipboard monitoring in the running Clipman instance.
clipman.exe --add "Some text" Adds the provided text directly to Clipman history.
clipman.exe --export "C:\Path\backup.clipdb" Exports the current Clipman database to a compressed .clipdb file. Existing files are not overwritten.
clipman.exe --import "C:\Path\backup.clipdb" Imports entries from a .clipdb, JSON, or text file into the current database without replacing existing entries.
clipman.exe --import-replace "C:\Path\backup.clipdb" Asks for confirmation, creates a timestamped backup of the current database, then replaces the current database with entries from the selected .clipdb, JSON, or text file.
clipman.exe --import-replace "C:\Path\backup.clipdb" --yes Runs the same replacement silently for scripts. A timestamped backup is still created first.
clipman.exe "C:\Path\Example.txt" Adds supported text-oriented file content to Clipman history. Examples include .txt, .log, .md, .csv, .json, .xml, .html, .ps1, .cmd, .bat, .ini, .ahk, .reg, and common source-code files.
