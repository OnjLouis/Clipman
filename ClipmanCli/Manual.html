Clipman CLI

Development preview: this build is available for practical testing and feedback. Command names, configuration details, and behavior may change before the first stable CLI release.

Clipman CLI is an additional command-line client for Clipman text history. It does not replace the Windows, macOS, Android, or iOS apps. It is intended for terminals, SSH sessions, scripts, headless systems, and users who prefer ordinary text commands.

Clipman CLI connects to Clipman Server and reads and writes the same encrypted text-history database as the other clients. Clipman Server requires a nonblank, preferably unique history password. File history, Secrets, local clipboard monitoring, and server administration remain in the relevant apps or server tools.

Quick Start

Choose the binary matching the current operating system and processor. It is standalone and does not require Go to be installed. You may rename that binary to clipman-cli or clipman-cli.exe and place it on the command search path.

Run initialization once for each configuration. The easiest method is to use the private importable connection file created by Clipman Server:

clipman-cli --password "history password" init --connection-file clipman-server-connection.clpconf

Initialization checks the server and, when the selected database already exists, verifies that the history password can decrypt it. If the selected database does not exist, Clipman cannot distinguish a new password from a mistyped password. It warns before any history is created.

Add and retrieve text through standard input and standard output:

printf '%s' 'Text from a terminal' | clipman-cli put
clipman-cli get
clipman-cli get --name deploy-command > command.txt

Commands

CommandPurpose
initConfigure and test a server profile. Supports connection files, token files, interactive secret entry, a machine name, and non-interactive setup.
statusCheck the server and selected history. Add --refresh to download and decrypt the database.
listList entries. Filter by group, search text, or kind; use --json or --porcelain for scripts.
getWrite one entry to standard output. Template variables resolve unless --raw is used.
putAdd UTF-8 text from standard input, --file, or --text. Standard input avoids exposing text in the process list.
rmDelete exactly one entry and create the same synchronization tombstone used by Clipman desktop.
pickShow a numbered accessible list on the controlling terminal and write the chosen entry only to standard output.
menuOpen the line-based history manager for viewing, outputting, deleting, and refreshing entries.
syncDownload and validate current server history. This stateless edition has no offline write queue.

Run clipman-cli --help or clipman-cli COMMAND --help for the current syntax. Global options must precede the command. Command options follow it.

Selecting Entries

get and rm accept a visible zero-based index, exact ID, exact name, or text search. Indices belong to the selected view and can change after another client updates history. Scripts should prefer exact IDs or names.

clipman-cli get 3
clipman-cli get --id 0123456789abcdef0123456789abcdef
clipman-cli get --search kubectl --first
clipman-cli rm --name old-command --yes

An ambiguous search is not treated as bulk deletion. Use a unique ID or a reviewed visible index.

Interactive Use

Run clipman-cli menu, or run clipman-cli without arguments in an interactive terminal. The line renderer avoids full-screen cursor movement and is suitable for screen readers.

  • Type an entry number to view it.
  • Type o, a space, and an entry number to write that entry to standard output and exit.
  • Type d, a space, and an entry number to delete it after confirmation.
  • Type r to refresh or q to quit.

pick presents one numbered choice and then exits. Prompts use the controlling terminal, so piped clip text remains separate.

Templates

Use put --template to create a template entry. get, pick, and menu output resolve the established Clipman date, time, time-zone, operating-system, and username variables. get --raw returns the stored text unchanged. Unknown variables remain unchanged.

clipman-cli put --text '{{year_full}}/{{month_num_padded}}/{{day_of_month_padded}}' --template --name today
clipman-cli get --kind templates --name today

Streams and Scripts

Successful get and pick commands write only clip text to standard output. Prompts and status messages do not enter the data stream. Text must be valid UTF-8 and may contain NUL bytes, whitespace, CRLF, LF, or no final newline.

--newline adds one LF only when the selected text does not already end in LF. --json provides structured output where documented.

Configuration

The default configuration file is ~/.clipman/config.toml on Linux and macOS, and %LOCALAPPDATA%\Clipman CLI\config.toml on Windows. CLIPMAN_HOME changes the data directory. CLIPMAN_CONFIG or --config selects a particular config file.

The nonblank history password is resolved from --password, CLIPMAN_PASSWORD, saved configuration when explicitly enabled, or a no-echo controlling-terminal prompt. This allows clip text to be read from standard input without consuming the password prompt.

Security

  • Encrypted CLIPDB2 data is authenticated before decryption or decompression.
  • Database, JSON, nesting, entry-count, and text-size limits are enforced.
  • Server tokens, passwords, full database IDs, and clip text are not written to diagnostic output.
  • On Windows, saved token and password values are protected for the current Windows user with DPAPI. On Linux and macOS, configuration is restricted to the owning user with directory mode 0700 and file mode 0600; explicitly saving a password there stores it in that owner-only file.
  • --password, --token, and --text can be visible to other software inspecting the process list. Prefer prompts, environment variables used carefully, token files, connection files, standard input, or --file.
  • clipman:// is a convenient name for plain HTTP. Use it only on loopback, a trusted private network, or a VPN. Use HTTPS across an untrusted network.
  • Keep the Clipman Server token private. Delete or secure its connection-details file after configuring clients.

Exit Codes

CodeMeaning
0Success
1Local input or output failure
2Usage, selection, or confirmation error
3Configuration, permission, or server-authentication error

5History password required, incorrect, or database authentication failed
6Requested entry or database not found
